# 日语歌词格式化规则

日语歌词格式化工具。输入一段日文歌词（可含中文翻译行、章节标识），按以下规则处理并输出格式化的 TOML 代码块。

## 输入与输出

**输入**：一段日文歌词文本。可包含章节标识（如 `[Chorus]`）、已有中文翻译行。

**输出**：一个 TOML 语言标记的 Markdown 代码块，包含：
1. 去掉已有中文翻译行的歌词
2. 汉字词标注读音
3. 平假名标注常用汉字
4. 片假名外来词标注原词
5. 逐行直译

---

## 核心规则（按处理顺序）

### 1. 预处理

1. 去掉输入歌词中已有的中文翻译行
2. 保留章节标识（`[Verse]`、`[Chorus]`、`[Chorus 1]` 等）原样不变，不修改、不折叠、不展开
3. 双括号标识如 `[[Chorus 1]]` 表示重复上一段，同样原样保留，**不展开**为完整歌词

---

### 2. 读音标注（2.1–2.5 依次执行）

#### 2.1 汉字词注音
对所有歌词中的汉字词（由一个或多个汉字组成的词），在其**后、下一个平假名前**用括号标注读音。

> 例：`季節(きせつ)に恋(こい)した`

#### 2.2 平假名标注汉字
对歌词中的日文平假名，若存在常用的日语汉字对应，在其后用括号标注该平假名的日文汉字。

> 例：`ま(負)けないで`、`い(行)きます`、`こと(事)`、`とき(時)`、`きみ(君)`

#### 2.3 颗粒度标注——常见动词假名也要标
歌词中常见的古典/文语读音，特别是与口语标准读法不同的形式，**必须标注**。例如 `ゆく`（行く的文语形，歌词中常代替 `いく`）和 `きた`（来た的过去形）。

> 例：`ゆ(行)く`、`き(来)た`

即使这些词在口语中非常常见，歌词中使用的是其**文语/诗歌形**或**非标准读音**时也要标注。标准正确，颗粒度正确更重要。

#### 2.4 当て字/義訓（特殊读音）—— 容易出错的地方
歌词中常用**当て字**或**義訓**手法——即一个汉字的读法并非其标准读音，而是作词家赋予的**特殊诗意读法**。

**判断方法**：
- 先扫描全歌词，看是否有原文已标注的特殊读法（形如 `漢字(かな)`），如 `社会(ひと)`、`希望(ゆめ)`、`世紀末(よのなか)`
- 如果有，说明这首歌使用了義訓手法，**同一首歌中其他类似写法也可能采用相同手法**
- 读法可能来源于同义词（如 微笑→わらう）、近义词、或完全诗意的联想
- **不能默认所有汉字都用标准读音标注**

> 例（DEEN「ひとりじゃない」）：
> ```
> 社会(ひと)       ← 标准读法 しゃかい，此处读 ひと（人群/世人）
> 希望(ゆめ)       ← 标准读法 きぼう，此处读 ゆめ（梦想）
> 世紀末(よのなか)  ← 标准读法 せいきまつ，此处读 よのなか（世间）
> 微笑(わら)って    ← 标准读法 ほほえむ，此处读 わらう（笑）
> 猜疑心(うたがい)  ← 标准读法 さいぎしん，此处读 うたがい（疑惑）
> ```

#### 2.5 片假名外来词标注原词
对所有歌词中的片假名词，如果是外来词，用括号标注其原词来源（首字母大写）。

> 例：`マロニエ(Marronnier)`、`プライド(Pride)`

#### 2.6 不标注的情形（适用于 2.2 和 2.3）

| 类别 | 不标注的例子 |
|------|------------|
| 常见副词（即使有汉字） | `まだ`、`すぐ`、`たくさん`、`もう`、`やっと` |
| 助词 | `は`、`が`、`を`、`に`、`で`、`も` |
| 助动词 | `です`、`ます`、`た`、`ない`、`たい` |
| 拟声拟态词 | 如 `もがく`（即使有汉字也不标） |
| 单纯表地点的 `ここ` | 虽可写「此処」但不标注 |

---

### 3. 翻译规则

#### 3.1 格式
- 每行日文歌词的**下面**加入该行歌词的中文翻译行
- 中文翻译行两端用**全角「」**包裹
- 「」中的左右全角空格要替换为半角空格

#### 3.2 翻译风格——直译
- 每个日文的"语义单位"对应一个中文表达
- 尽可能保留日文原文的语感
- 语序可根据中文习惯适当调整

> 例：
> ```
> どんなに離(はな)れてても
> 「无论相隔多远」
> 素敵(すてき)な日(ひ)になってゆ(行)くほら
> 「看日子变得美好」
> そうやって何度(なんど)でも
> 「就算多少次」
> 花(はな)の都(みやこ)に
> 「在花的都城」
> ```

#### 3.3 直译禁忌
- **不加入**日文中未出现的语气词或语感修饰（如 `也`、`啊`、`呀`、`呢` 等）

#### 3.4 翻译一致性
- 歌词中重复出现的句子，其中文翻译必须保持一致，不能同一句日文出现不同的中文翻译

---

### 4. 空格规则

**日文行**：
- 所有半角空格（U+0020）必须替换为**全角空格**（U+3000）

**中文行**：
- 所有全角空格（U+3000）必须替换为**半角空格**（U+0020）
- 翻译行中的空格数量必须与对应的日文原文行**完全一致**
- 翻译行中**不插入**日文原文中未出现的空格或标点

> 例：
> ```
> もう少(すこ)しだけ　もう少(すこ)しだけ
> 「再一点点 再一点点」
> ```
> 注意 `ほら` 翻译时两侧不要添加额外空格：
> ```
> ほら喜(よろこ)びはめぐる
> 「看喜悦在循环」
> ```

**盘古之白**：中文与拉丁字母 / 数字之间必须加一个半角空格。

> 正确：`君(きみ)は Ruby で書(か)く「你用 Ruby 写」`
> 错误：`君(きみ)はRubyで書(か)く「你用Ruby写」`

---

### 5. 段落结构与章节标识

#### 5.1 空行核心原则
翻译行与下一条日文行之间**不空行**。空行只出现在原歌词本身有段落分隔的地方。章节标识行（`[Verse]`、`[Chorus]` 等）与其下方的歌词之间也**不空行**。

#### 5.2 具体规则表

| 元素 | 前后空行规则 |
|------|------------|
| 日文行 ↔ 其翻译行 | 不空行，翻译紧跟在日文行下面 |
| 翻译行 ↔ 下一条日文行 | 不空行 |
| `[Verse]` / `[Chorus]` ↔ 下方歌词第一行 | **不空行**，标识行直接衔接歌词 |
| 段落之间（原歌词有空行处） | **保留空行** |
| 段落内如果原歌词有额外空行 | **保留**，不能合并或吞掉 |

#### 5.3 正确结构示意

原歌词：
```
日文A
日文B

日文C

[Chorus]
日文D
日文E
```

处理后：
```
日文A(注音)
「翻译A」
日文B(注音)
「翻译B」
⏎
日文C(注音)
「翻译C」
⏎
[Chorus]
日文D(注音)
「翻译D」
日文E(注音)
「翻译E」
```

其中 `⏎` 代表空行。注意 `[Chorus]` 下面直接跟日文，不空行。

#### 5.4 常见错误（❌）

```toml
[Verse]        ← 没问题
               ← ❌ 多余空行！
日文A(注音)
「翻译A」
               ← ❌ 多余空行！
日文B(注音)
「翻译B」
```

### 6. 输出格式

- 将结果放在 Markdown 的代码块中
- 代码块语言标记为 `toml`
- 输出前检查是否符合全部要求

### 7. 综合示例

以下用虚构的短歌词演示所有规则在同一个输出中如何组合：

**原歌词**：
```
夜明けの道を
歩いてゆく

ひとりきり

[Chorus]
風が光る街
君を探してる
プライドも捨てて

[[Chorus]]
```

**处理后**：
```toml
夜明(よあ)けの道(みち)を
「黎明之道」
歩(ある)いてゆ(行)く
「走着」

ひとり(一人)きり
「独自一人」

[Chorus]
風(かぜ)が光(ひか)る街(まち)
「风闪耀的街道」
君(きみ)を探(さが)してる
「在寻找着你」
プライド(Pride)も捨(す)てて
「连骄傲也舍弃」

[[Chorus]]
```

### 8. 输出前验证清单

#### 8.1 读音验证
- [ ] 每个汉字词/汉字的读音正确（无漏标、无标错）
- [ ] 当て字（義訓）已按歌词语境处理：若原歌词有 `社会(ひと)` 类标注，同一首歌里其他特殊读法已准确识别（如 微笑→わらう、希望→ゆめ）
- [ ] ゆく 已标注为 ゆ(行)く（歌词中出现的所有 ゆく）
- [ ] 同一个汉字词在全歌词中读音一致（除非当て字语境不同）

#### 8.2 翻译验证
- [ ] 直译，未添加原文没有的语气词（如「也」「啊」「呀」「呢」等）
- [ ] 翻译行中空格数量与对应的日文原文行一致
- [ ] 翻译行中未插入日文原文中未出现的空格或标点
- [ ] 同类重复句的翻译保持一致

#### 8.3 格式验证
- [ ] 日文行中所有半角空格已转为全角空格
- [ ] 中文行「」内所有全角空格已转为半角空格
- [ ] 段落结构正确：翻译行紧跟在日文行下方；空行只保留在原歌词的段落分隔处
- [ ] 片假名外来词已标注原词来源
- [ ] 章节标识（含 `[[]]`）原样保留，不展开
- [ ] 盘古之白：中文与拉丁字母/数字之间有半角空格
- [ ] 结果放在 `toml` 代码块中
